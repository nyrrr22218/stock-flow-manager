## 🛠 解決したかった課題とアプローチ

### 1. 現場のストレスをなくすシステム設計

在庫管理は本来、一元管理し誰でも把握できる状態が好ましいです。
目視や手作業での確認作業は人的資源の浪費につながります。
また、仕様の際の読み込み待ちを排除するため、以下の最適化を行いました。

- **Next.js 16 (App Router) × Prisma 7.2**: サーバーサイドでのデータ取得を最適化し、First Contentful Paint 0.3秒という極めて高いレスポンス性能を実現。
- **データフローの純化**: 通信経路を精査し、不要なAPIリクエストを排除。最小限ので動作する構築にしました。

### 2. 「型安全」によるデータの正確性担保

手作業による在庫確認の漏れや誤入力を防ぐため、システムレベルでのガードを徹底しています。

- **Zod × TypeScript**: 入力データの型定義をフロントエンドからバックエンドまで一貫させ、バリデーションを厳格化。不正なデータがDBに混入するリスクを抑えています。
- **MUI X Data Grid**: 大規模な在庫データでも、正確かつ直感的に編集・フィルタリングできるUI。

### 3. 意思決定を支える可視化

単に数値を記録するだけでなく、Rechartsを用い在庫状況を視覚的に把握でき、迅速な生産計画を支援します。
