## 🛠 解決したかった課題とアプローチ

### 1. 現場の「待ち」をなくすシステム設計
在庫管理は一分一秒を争う現場で行われます。従来のシステムにありがちな「読み込み待ち」を排除するため、以下の最適化を行いました。
- **Next.js 16 (App Router) × Prisma 7.2**: サーバーサイドでのデータ取得を最適化し、First Contentful Paint 0.3秒という極めて高いレスポンス性能を実現。
- **データフローの純化**: 通信経路を精査し、不要なAPIリクエストやXHRの重複を排除。最小限のペイロードで動作するアーキテクチャを構築しました。

### 2. 「型安全」によるデータの正確性担保
手作業による在庫確認の漏れや誤入力を防ぐため、システムレベルでのガードを徹底しています。
- **Zod × TypeScript**: 入力データの型定義をフロントエンドからバックエンドまで一貫させ、バリデーションを厳格化。不正なデータがDBに混入するリスクをゼロに抑えています。
- **MUI X Data Grid**: 大規模な在庫データでも、正確かつ直感的に編集・フィルタリングできるUIを提供。

### 3. 意思決定を支える可視化
単に数値を記録するだけでなく、Rechartsを用いたデータビジュアライゼーションを実装。在庫の推移を視覚的に把握でき、迅速な発注判断を支援します。
